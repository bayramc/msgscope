# Sample files

These are **tiny, synthetic, content-free** `.msg` files used for demos and as a
smoke test. They contain no personal data. They were generated by the in-repo
CFB writer (`tests/fixtures/`), not exported from Outlook.

| File | Purpose | Expected result |
|------|---------|-----------------|
| `good_minimal.msg` | A well-formed message. | No indicators. |
| `synthetic_orphan.msg` | The same message plus one unindexed `__substg1.0_` stream. | One `ORPHAN_SUBSTG` (high) indicator. |

Try them:

```console
msgscope inspect sample/good_minimal.msg
msgscope inspect sample/synthetic_orphan.msg
msgscope inspect sample/synthetic_orphan.msg --format json --reproducible
```

## Real-world corpus

Real known-good `.msg` files (saved from Outlook) and diverse known-synthetic
files (from third-party generators) are **not** committed — they may carry PII
and licensing constraints. Keep them locally under `corpus/` (git-ignored). See
[CONTRIBUTING.md](../CONTRIBUTING.md) for how to source and scrub them.
